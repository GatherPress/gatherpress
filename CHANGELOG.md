# Changelog

All notable changes to this project will be documented in this file.

The format is based on [Keep a Changelog](https://keepachangelog.com/en/1.1.0/),
and this project adheres to [Semantic Versioning](https://semver.org/spec/v2.0.0.html).

Pending entries for the next release live as individual files under
[`.github/changelog/`](.github/changelog/) and get rolled up into a new
version section by `composer changelog:write` at release time.

## [0.33.3] - 2026-02-16

### Fixed
- Protect GatherPress meta from the Custom Fields panel to prevent stale data overwrites on event save. [#1383]

## [0.33.2] - 2026-02-08

### Fixed
- Fix "not allowed to edit custom field" error when publishing events. [#1375]

## [0.33.1] - 2026-01-30

### Fixed
- Fix `.ics` downloads on sites with custom permalinks, auto-flush rewrite rules on plugin update, and resolve add-to-calendar block context inside Query Loops. [#1356]

## [0.33.0] - 2026-01-10

### Added
- New `gatherpress/event-query` block variation of `core/query` so theme authors can build event-aware loops from any template. [#962]
- New `gatherpress/rsvp-form` block with token-based authentication, anonymous submissions, guest counts, and configurable form-field blocks. [#1106] [#1116] [#1168] [#1179]
- New `gatherpress/add-to-calendar` block with cleaner architecture, replacing the legacy attendee-list-only Add to Calendar markup. [#1068]
- Block Guard feature for query-loop integration so editor previews stay interactive without breaking save flow. [#1159]
- Auto-generated developer hook docs under `docs/developer/hooks/`, regenerated by CI on every push to `develop`. [#1135]
- Display options for the Event Date block (date format, time format, separator). [#1155]
- Filter `gatherpress_default_duration_options` to customize the duration dropdown options. [#882]
- RSS feed for past events. [#1189]
- Loading animation while the RSVP API is in flight. [#1208]
- GDPR-compliant opt-in filter for event notifications. [#1225]
- Improved security policy with detailed reporting guidelines. [#1321]

### Changed
- Refactor RSVP form processing into a centralized helper class; UI polish across the RSVP block family. [#1224]
- Refactor CSS class naming across blocks to a consistent component pattern. [#1183]
- Improve autoloader flexibility by removing hardcoded directory mappings. [#1199]
- Tie submitted RSVP emails to existing users when possible; fix comment approval flow. [#1213]
- Add support for WordPress 6.9. [#1236]
- Add support for PHP 8.4 in the test matrix. [#1147]
- Move build directory out of the repository in favor of release artifacts. [#1066]
- Update `register_setting` calls to use sanitize callbacks. [#1074]
- Replace deprecated WordPress admin notice markup with `wp_admin_notice()`. [#1080]
- Simplify rewrite rules flushing by leveraging WordPress core's mechanism. [#1231]
- Add gesture-handling library so Leaflet maps don't hijack page scroll. [#1089]
- Update RSVP column on the admin events list. [#1237]
- Add post-type-aware event validation and past-event handling for the RSVP form visibility system. [#1239]
- Multisite test coverage added to the suite. [#1291]

### Fixed
- Resolve translation-loading error by moving GP core function to the `init` hook. [#1055]
- Allow multiple `gatherpress/venue` blocks per event. [#1095]
- Fix critical error when WPForms Pro is also installed. [#1118]
- Fix Block Guard behavior in FSE templates. [#1119]
- Fix Block Guard inside query-loop blocks. [#1159]
- Fix RSVP block dimming, focus trap, and button-block targeting. [#1244] [#1256] [#1269]
- Restrict the RSVP "view" filter to RSVP comment types so counts and the "Mine" filter are correct. [#1133]
- Prevent generic comment notifications from being sent to event authors when an RSVP comment is left. [#1218]
- Honor WordPress comment privacy filters on RSVP inserts. [#1470]
- Fix date-picker relative-mode behavior so duration offset is maintained when the start date changes. [#1219]
- Fix Event Query block toggle on hosts where `array_filter` was stripping `0` values. [#1235]
- Fix Moment Timezone error for sites configured with a Manual UTC Offset. [#1336]
- Fix UI/UX bug where dropdown children were not auto-expanding on parent select. [#1333]
- Fix hardcoded comment-type assumptions so third-party comment plugins keep working. [#1226]
- Fix `extract-wp-hooks` action when no new hooks are present. [#1230]
- Don't ship the `.wordpress-org` directory in plugin releases. [#1104]

### Removed
- Remove `window.GatherPress` global in favor of editor settings, interactivity state, and data stores. [#1465]

## [0.27.0] - 2024-01-22

Initial public release. Represents 18+ months of pre-1.0 development and ships the core feature set:

### Added
- Custom post types `gatherpress_event` and `gatherpress_venue` with REST API support.
- Event datetime handling with timezone support and a dedicated `gatherpress_events` database table.
- RSVP system with attending / not-attending / waiting-list states, max-attendance limits, and an attendees list.
- Venue management with name, address, phone, website, and embedded map.
- Online-event link handling (gated on RSVP state).
- "Add to calendar" links covering Google, iCal, Outlook, and Yahoo.
- Email functionality for site organizers to message attendees.
- Block library: Event Date, Venue, Online Event, Add to Calendar, Event List, Attendance Selector, Attendees, Modal, Dropdown.
- Event archive pages for upcoming and past events with configurable archive page assignments.
- Topic taxonomy for events.
- Settings UI under the Events admin menu, including roles and credits.
- WP-CLI integration.
- PHP 8.0 / 8.1 / 8.2 / 8.3 compatibility matrix.
- Initial unit test suite with code coverage via SonarCloud.
- Multilingual screenshots and i18n scaffolding.
